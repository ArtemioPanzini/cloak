Эта страница позволяет проверить ключевые свойства решения без чтения всей
документации и исходного кода.

## 1. Чистая установка

```bash
npm ci
```

Требуется Node.js 20 или новее. Установка использует публичный npm registry и
закреплённый `package-lock.json`.

## 2. Автоматические проверки

```bash
npm test
npm run typecheck
npm run build
```

Ожидаемый baseline до добавления новых contract assertions — 4 test files и
19 уникальных тестов. Production build компилирует только `src/` и не создаёт
копии тестов в `dist/tests`.

## 3. Запуск приложения

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000). Страница должна
содержать только dropdown штатов и кнопку `Submit`.

## 4. Проверка обычного submit

1. Выберите штат.
2. Подождите около секунды.
3. Нажмите `Submit`.

При согласованных browser/server signals demo-конфигурация направляет на
`/demo/offer`. Endpoint `POST /api/decision` возвращает только URL как
`text/plain`; score и причины клиенту не раскрываются.

## 5. Проверка no-JS flow

Отключите JavaScript или отправьте HTML-форму напрямую. `POST /submit`
возвращает `303 Location`. Hidden page token сохраняется, но client telemetry
отсутствует, поэтому coverage обычно недостаточна и demo-решение ведёт на
`/demo/whitepage`.

## 6. Проверка audit

```bash
npm run audit:tail -- 2
```

Последние решения находятся в `data/audit.jsonl`. Запись включает входные
данные, нормализованные сигналы, history snapshot, adjustments, итоговые score,
decision, destination URL и primary reason.
